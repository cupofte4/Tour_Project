Chapter 01–02

Luồng Khởi Động (Backend + Frontend)

Backend Startup Sequence

Tầng 1 — Security Config Check

Kiểm tra JWT_SECRET, thông tin kết nối DB (DB_HOST, DB_USER), và khóa mã hóa (nếu dùng) đã được cấu hình đúng. Từ chối khởi động nếu thiếu biến môi trường quan trọng.

Tầng 2 — Database Connection

Kết nối MySQL qua SQLAlchemy engine. Thực hiện ping/health-check và cấu hình connection pool.

Tầng 3 — Migration + Seeding

Chạy migration (Alembic) để đồng bộ schema. Tạo tài khoản admin mặc định (nếu chưa có) và dữ liệu mẫu Locations cho demo.

Tầng 4 — Indexing

Tạo index cho truy vấn gần vị trí: composite index cho (Latitude, Longitude), index cho Favorites(user_id), và index tìm kiếm Locations(Name). (Tối ưu: bounding box trước, rồi Haversine ở app layer.)

Gắn Routers → Server Ready

auth, locations, favorites, audio, admin — prefix /api. Bật CORS cho frontend domain và bật OpenAPI docs cho môi trường dev.

Frontend Startup Flow

SW Register → React Router (catch-all) → Splash Screen → Chọn ngôn ngữ → Parallel Load

🛰️ GPS

waitForPosition() — xin quyền vị trí, lấy tọa độ lần đầu (high accuracy), timeout 10s. Nếu từ chối → chuyển sang chế độ duyệt theo danh sách.

💾 Offline Data

getOfflineLocations(lang) — lấy từ IndexedDB để hiện tức thì (danh sách + chi tiết đã cache).

🌐 Online Data

loadLocations(lang) → gọi API → saveLocations() → cập nhật IndexedDB + cache ảnh/audio.

⭐ Nearby Set

Lọc địa điểm trong bán kính (ví dụ 1–2km) để ưu tiên hiển thị và chuẩn bị audio (prefetch) cho trải nghiệm mượt.

📦 Offline Pack

Tùy chọn: người dùng tải sẵn gói dữ liệu (danh sách/ảnh/audio) cho một khu vực để dùng khi mất mạng.

Chapter 04

GPS + Geofence + Audio Narration

Hệ thống xử lý các vấn đề thực tế: nhiễu GPS (debounce), tối ưu pin (throttle), và phát audio mượt mà ngay cả khi offline. Điều kiện kích hoạt: người dùng cách địa điểm < 50m.

4 Giai Đoạn Xử Lý

Giai đoạn 1 — GPS Collection (LocationService)

watchPosition() liên tục → throttle 5s → tránh tính Haversine/vẽ WebGL/sort địa điểm liên tục. Cập nhật chấm xanh trên map.

Giai đoạn 2 — Zone Detection (GeofenceEngine)

checkGeofences(pos): tính khoảng cách bằng Haversine (meters). Nếu distance < 50m → pendingEntries. Khi ra khỏi vùng đã trigger → cooldown để tránh phát lặp liên tục.

Giai đoạn 3 — Audio Decision (Heartbeat 1s)

reconcileLoop mỗi 1s: xác nhận ENTER sau debounce → chọn địa điểm ưu tiên (khoảng cách + trạng thái audio) → đẩy NarrationEngine.

Giai đoạn 4 — Audio Playback (4-Tier Hybrid)

NarrationEngine → AudioQueueManager (single-slot priority queue) → playWithFallback()

Audio Playback Fallback

Tier 1
Uploaded / Pre-recorded Audio Ưu tiên phát file audio đã được admin upload cho đúng ngôn ngữ (VI/EN/ZH/DE). Nếu đã cache offline → phát tức thì.
Tier 2
Server-side TTS (Optional) Nếu thiếu audio, backend có thể sinh audio TTS từ nội dung (theo ngôn ngữ đã chọn) và lưu để tái sử dụng (phục vụ offline pack).
Tier 3
On-device Speech Synthesis window.speechSynthesis — fallback cuối cùng khi offline và không có audio file; đảm bảo luôn “có tiếng nói”.
Background Prefetch: Khi người dùng di chuyển, app ưu tiên cache audio/ảnh của các địa điểm gần nhất (ví dụ trong 500m) để giảm độ trễ lúc vào geofence. Cơ chế throttle/backoff để tránh tải quá mức.

Chapter 05

Module Content — Quản Lý Địa Điểm

GET /locations

Lấy danh sách địa điểm theo ngôn ngữ (lang=vi|en|zh|de). Trả về name/summary/lat/lng, ảnh đại diện và trạng thái audio để frontend hiển thị và chuẩn bị auto-play.

GET /locations/nearby

Nhận lat, lng, radius (m) → trả về các địa điểm gần. Server có thể lọc sơ bộ theo bounding box, frontend dùng Haversine để quyết định trigger < 50m.

POST /admin/locations

Admin tạo địa điểm: lưu metadata + nội dung thuyết minh VI/EN/ZH/DE. Audio có thể upload theo từng ngôn ngữ hoặc dùng TTS (tùy chọn) để sinh audio.

DELETE /admin/locations/{id}

Xóa địa điểm và dọn dữ liệu liên quan (favorites, audio files, cache keys). Đảm bảo không còn file rác trên storage.

Content Fallback (Ngôn Ngữ)

Tier 1
Ngôn ngữ yêu cầu VI/EN/ZH/DE: ưu tiên nội dung và audio đúng ngôn ngữ người dùng đã chọn.
Tier 2
English Fallback (EN) Nếu thiếu nội dung của ngôn ngữ mục tiêu, fallback sang EN (phổ biến, dễ chuẩn hóa).
Tier 3
Nguyên bản tiếng Việt (VI) Lớp phòng thủ cuối cho nội dung. Nếu thiếu audio, hệ thống có thể dùng TTS on-device để đảm bảo phát được.
Offline Caching: Audio/ảnh có thể được cache theo ngôn ngữ và theo địa điểm. Khi mất mạng, app vẫn xem được danh sách và vẫn phát được audio đã tải trước đó.

Chapter 06

Module Audio / TTS

Audio Assets (Uploaded)

  • Admin upload audio thuyết minh theo (location_id, lang): vi/en/zh/de.
  • Lưu metadata: duration, checksum, updated_at để phục vụ cache-bust và đồng bộ offline.
  • Quy ước định dạng: MP3/AAC, bitrate phù hợp mobile để giảm dung lượng offline.

Optional Server-side TTS

  • Khi thiếu audio, admin có thể yêu cầu backend sinh TTS từ text (theo lang) và lưu thành audio asset.
  • Trạng thái: queued → processing → completed/failed; có log lỗi để xử lý lại.
  • Mục tiêu: đảm bảo đủ audio cho chế độ offline, không bắt buộc cho phiên bản MVP.

Frontend Audio Player

  • Tự động mở mini-player khi ENTER geofence và có audio tương ứng.
  • Điều khiển: play/pause, stop, auto-skip khi ra khỏi vùng hoặc chuyển sang địa điểm ưu tiên hơn.
  • Fallback: nếu không có audio file → dùng window.speechSynthesis.

Offline Audio Caching

  • Service Worker cache audio theo ngôn ngữ để giới hạn dung lượng.
  • Prefetch theo “nearby set” và/hoặc theo danh sách yêu thích.
  • Cache-bust theo updated_at hoặc checksum khi admin cập nhật nội dung.

Audio Generation Pipeline

Location Text → Admin Upload → Audio Asset (per lang) → Cache Metadata → Service Worker Cache → Offline Ready → Auto-play on ENTER

Chapter 07

Authentication & RBAC

🔐 JWT Authentication

  • POST /api/auth/register, POST /api/auth/login
  • access_token: JWT (ví dụ 30 phút) để gọi API
  • refresh_token (tùy chọn): kéo dài phiên đăng nhập (ví dụ 7 ngày)
  • Mật khẩu được hash bằng bcrypt; không lưu plain text

🛡️ RBAC (user/manager/admin)

  • Role nằm trong bảng Users.Role: user, manager, admin
  • User: xem bản đồ, xem địa điểm, nghe audio, quản lý yêu thích
  • Manager: quản lý các địa điểm được gán (CRUD nội dung), quản lý thuyết minh (TextVi/En/Zh/De), xem thống kê theo thời gian
  • Admin: CRUD Locations, quản lý Users/Managers, phân công địa điểm cho manager
  • API guard: require_role("admin") cho /api/admin/*, require_role("manager") cho /api/manager/*

Role Priority Permissions

Account Safety: Admin có thể khóa tài khoản (IsLocked=true). Backend áp dụng rate limit cho login để giảm brute-force và log các sự kiện đăng nhập phục vụ kiểm tra.

Chapter 08

Module Localization — Đa Ngôn Ngữ Thông Minh

Flow 1 — Language Select

User chọn ngôn ngữ thuyết minh: VI / EN / ZH / DE. Frontend lưu lang vào LocalStorage và dùng tham số ?lang= khi gọi API.

Flow 2 — Content Fallback

Nếu thiếu nội dung/audio của ngôn ngữ mục tiêu: fallback theo chuỗi Target → EN → VI. UI hiển thị rõ nhãn ngôn ngữ đang phát để tránh gây nhầm lẫn.

Flow 3 — Admin Localization

Admin chỉnh sửa nội dung thuyết minh theo từng ngôn ngữ (TextVi/TextEn/TextZh/TextDe) và upload audio tương ứng. AI-assist (tùy chọn) hỗ trợ viết nháp/dịch, nhưng admin chịu trách nhiệm duyệt.

Flow 4 — Offline Cache by Language

IndexedDB lưu Locations theo lang. Service Worker cache audio theo ngôn ngữ để tối ưu dung lượng và cho phép chuyển ngôn ngữ mượt khi online.

Nguyên tắc chất lượng: Nội dung đa ngôn ngữ phải nhất quán giữa text và audio. Hệ thống ưu tiên “đúng ngôn ngữ” nhưng luôn có fallback để đảm bảo trải nghiệm không bị đứt quãng khi thiếu dữ liệu hoặc khi offline.

Chapter 09

Offline / PWA — 4 Lớp Phòng Ngự

Lớp 1 — Service Worker Strategies

sw.js (Workbox-based). Chiến lược cache theo loại tài nguyên:

  • Locations data: NetworkFirst (timeout 8s), cache locations-api-cache, TTL 15 phút
  • Audio: CacheFirst theo ngôn ngữ, cache audio-cache-lang-{lang}
  • Images: CacheFirst, 30 ngày, max 50
  • App shell: Precache (HTML/CSS/JS) để app mở được khi offline

Lớp 2 — Language Sharding + LRU

Mỗi ngôn ngữ = ngăn kéo riêng. URL param ?l={lang} → SW phân loại.

  • Max 300 files / ngôn ngữ
  • Max 3 ngôn ngữ đồng thời
  • LRU eviction: xóa lang ít dùng nhất

Lớp 3 — IndexedDB (TourGuideDB v1)

Store locations_by_lang, key = "{lang}:{locationId}".

  • Offline Fallback: target → en → vi
  • UI hiển thị ngôn ngữ đang dùng để tránh nhầm lẫn
  • User không bao giờ thấy màn hình trống

Lớp 4 — Offline Packs

Offline mode tập trung vào nội dung + audio (PWA), đảm bảo ứng dụng vẫn dùng được khi mất mạng.

  • IndexedDB lưu danh sách/chi tiết Locations theo ngôn ngữ
  • Cache API lưu ảnh và audio đã nghe / đã prefetch
  • Chiến lược ưu tiên: Cache → Network (khi online)
  • Giới hạn dung lượng theo ngôn ngữ, tự dọn dẹp khi thiếu bộ nhớ

SW Message Protocol

Message Từ Mục đích
SET_ACTIVE_LANGUAGE Frontend Service Worker ưu tiên giữ cache của ngôn ngữ hiện tại (VI/EN/ZH/DE) và dọn cache ngôn ngữ ít dùng khi hết dung lượng.
PREFETCH_NEARBY Frontend Yêu cầu SW prefetch audio/ảnh của các địa điểm gần nhất để giảm độ trễ khi ENTER geofence.
CLEAR_OFFLINE_CACHE Settings Xóa dữ liệu offline: audio/image cache và IndexedDB (theo ngôn ngữ hoặc toàn bộ).
SYNC_FAVORITES Frontend Khi online trở lại, đồng bộ danh sách yêu thích và lịch sử phát lên server.
OFFLINE_STATUS Service Worker Thông báo trạng thái cache/offline (dung lượng, ngôn ngữ đang giữ, số lượng audio đã cache) để UI hiển thị minh bạch.
Edge Case — Disk Full: Khi gặp QuotaExceededError, Service Worker sẽ dọn cache cũ (theo LRU) và ưu tiên giữ audio của ngôn ngữ đang dùng và các địa điểm đã được nghe gần đây.

Chapter 10

Map System — Leaflet Map

Frontend sử dụng Leaflet Map để hiển thị vị trí người dùng, Leaflet marker địa điểm, và dẫn hướng cơ bản. Khi offline, hệ thống vẫn giữ được danh sách/chi tiết/audio đã cache; bản đồ có thể hiển thị ở chế độ hạn chế.

Chuỗi Liên Kết 4 Tầng (End-to-End)

Tầng Thành phần chính Vai trò
1. Map Initialization Leaflet.js + OpenStreetMap Tiles Khởi tạo bản đồ, style cơ bản, và các control (zoom, my-location).
2. Data Layer GET /api/locations, /nearby Lấy danh sách địa điểm, Leaflet marker data, và chi tiết theo ngôn ngữ.
3. Location Tracking Geolocation watchPosition Cập nhật vị trí theo throttle; chạy GeofenceEngine để quyết định trigger audio < 50m.
4. Offline Fallback IndexedDB + Service Worker cache Khi offline: vẫn hiển thị danh sách/chi tiết/audio đã cache; bản đồ ở chế độ hạn chế hoặc tạm ẩn.

3 Chế Độ Bản Đồ

🌐 Online (Default)

Leaflet Map đầy đủ: vị trí người dùng, Leaflet marker, tìm kiếm cơ bản và chỉ đường (tùy phạm vi).

📴 Offline (Limited)

Khi không có mạng: ưu tiên nội dung (Locations + Audio) đã cache. Bản đồ hiển thị hạn chế (ví dụ vị trí hiện tại + danh sách gần) tùy khả năng tải tài nguyên.

🔄 Hybrid

Online map + offline content/audio: ngay cả mạng yếu, audio đã cache vẫn phát được; dữ liệu địa điểm có thể được cập nhật dần khi online.

Chapter 11

Admin Dashboard (CMS)

Admin là nơi quản trị toàn bộ dữ liệu địa điểm và phân quyền: nội dung thuyết minh, tọa độ, ảnh, audio theo ngôn ngữ, quản lý người dùng/manager và phân công địa điểm cho manager. Mục tiêu là đảm bảo dữ liệu “đúng, đủ, nhất quán” để auto-play hoạt động tin cậy.

📍 Location Management

  • CRUD Locations: tên, mô tả, địa chỉ, lat/lng, ảnh
  • Quản lý text đa ngôn ngữ: TextVi/TextEn/TextZh/TextDe
  • Quản lý geofence: cấu hình bán kính (mặc định 50m), bật/tắt auto-play cho từng địa điểm
  • Upload audio theo ngôn ngữ và kiểm tra nghe thử trước khi publish

👤 User Management

  • Danh sách Users, xem trạng thái và role (user/manager/admin)
  • Khóa/mở khóa tài khoản: IsLocked
  • Reset mật khẩu (theo quy trình nội bộ) và log thao tác quản trị
  • Quản lý manager: cấp/thu role manager, kiểm soát truy cập theo doanh nghiệp

🔗 Assignment Management

  • Phân công địa điểm cho manager: bảng LocationManagerAssignments
  • Cho phép 1 manager quản lý nhiều địa điểm; tránh trùng phân công bằng unique key
  • Tra cứu nhanh: manager → danh sách địa điểm đã gán / địa điểm khả dụng
  • Mục tiêu: tách quyền “quản trị hệ thống” (admin) và “quản trị nội dung theo doanh nghiệp” (manager)

🤖 AI-assisted Content Writing (Optional)

  • Gợi ý cấu trúc bài thuyết minh: giới thiệu, lịch sử, điểm nổi bật, hướng dẫn tham quan
  • Hỗ trợ dịch thuật VI ↔ EN/ZH/DE và chuẩn hóa thuật ngữ
  • Nguyên tắc: không bịa đặt; admin phải kiểm duyệt trước khi publish
  • Ghi log prompt/output để truy vết và cải thiện chất lượng

Admin Publishing Flow

Admin login → Tạo/Update Location → Upload audio (VI/EN/ZH/DE) → Publish → User app sync → ready

sequenceDiagram
    participant U as User/Browser
    participant SW as Service Worker
    participant FE as React Frontend
    participant MAP as Leaflet Map
    participant GE as GeofenceEngine
    participant AP as FastAPI Backend
    participant DB as MySQL
    participant AU as Audio Player

    U->>SW: Mở app → SW register
    FE->>U: Chọn ngôn ngữ (VI/EN/ZH/DE)
    FE->>SW: getOfflineLocations(lang) (IndexedDB)
    SW-->>FE: Cached list + details
    FE->>AP: GET /api/locations?lang=en
    AP->>DB: Query Locations (+ audio metadata)
    DB-->>AP: Rows
    AP-->>FE: Locations response
    FE->>MAP: Render map + markers
    FE->>GE: startTracking (watchPosition)

    loop Every 5s (GPS Throttle)
        U->>GE: GPS position update
        GE->>GE: Haversine distance to Locations
        GE->>GE: ENTER when distance < 50m (debounce)
        GE->>AU: Auto-play narration for best location
    end

    alt Audio file available (preferred)
        AU->>SW: fetch audio asset (CacheFirst)
        SW-->>AU: MP3 from cache or network
    else Fallback TTS
        AU->>U: window.speechSynthesis (offline safe)
    end
    AU-->>AP: POST /api/history (location_id, lang, timestamp)
    AP->>DB: Insert PlayHistory
    AP-->>AU: 200 OK
    AU-->>FE: onFinished → close popup
