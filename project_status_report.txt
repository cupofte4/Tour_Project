3. BACKEND HIEN TAI

3.1. Nen tang

Backend dang dung:
- .NET 8
- ASP.NET Core Web API
- EF Core 8
- Npgsql.EntityFrameworkCore.PostgreSQL
- Swagger

Program.cs cho thay backend da duoc wiring that:
- AddApplication()
- AddInfrastructure(builder.Configuration)
- AddControllers()
- AddEndpointsApiExplorer()
- AddSwaggerGen()
- Tu dong migrate database khi startup neu DB la relational
- Tu dong chay IRemoteLocationSyncService.SyncAsync() khi app khoi dong

Y nghia:
- Backend khong con o muc skeleton.
- Database schema duoc quan ly bang migration.
- Du lieu stall dang duoc dong bo tu nguon ngoai/snapshot, khong phai seed tay don gian.

3.2. API endpoint dang co

Controller chinh hien tai la StallsController, gom:
- GET /api/stalls
- GET /api/stalls/{id}
- GET /api/stalls/nearby?lat={x}&lng={y}&radius={r}
- GET /api/stalls/{id}/translation?lang=...
- POST /api/stalls/sync/locations

Ngoai ra endpoint goc "/" tra ve:
- Name = "Vinh Khanh Guide API"
- Status = "Ready"

Danh gia:
- Phan read API cho danh sach, chi tiet, nearby va translation da co.
- Da co endpoint sync de nap/cap nhat du lieu.
- API hop voi mot mobile app tham quan/du lich lay noi dung tu backend.

3.3. Domain va application logic

Phan backend hien da co nhieu logic nghiep vu hon ban CRUD co ban:
- Stall va StallTranslation da xuat hien ro trong Domain/Application.
- Translation service co ho tro vi, en, zh, de.
- RemoteLocationSyncService xu ly dong bo stalls va translations.
- Khi translation khong co san, service co fallback ve noi dung tieng Viet.

Tinh trang rat dang chu y:
- Sync service dang khong chi upsert stall ma con doi chieu translations dang ton tai va xoa translation khong con nam trong nguon mong muon doi voi cac ngon ngu duoc ho tro.
- Day la dau hieu cho thay he thong da co "vong doi du lieu" ro hon, khong chi them moi.
